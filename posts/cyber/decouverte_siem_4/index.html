Introduction

Dans ce nouveau post, comme promis dans le dernier post, je vais vous montrer la vraie puissance de Wazuh et comment est-ce que Wazuh peut nous aider a ameliorer la securite de notre infrastructure, avec bien entendu avant de faire ça, je vais installer Sysmon sur Windows (j’expliquerais plus tard ce que c’est), ajouter quelques rules Wazuh pour ameliorer la capacite de detection de Wazuh. Tout cela est plus dans ce post !

Nouvelles Rules

Dans cette grande partie on va faire deux choses principalement, on va installer d’abord Sysmon sur nos systemes Windows, egalement on va revisiter nos configurations de groupes pour l’ameliorer et optimiser, et une fois que nous aurons fait ça, j’importerais des nouvelles regles sur notre Wazuh-Server pour avoir une meilleure detections des possibles menaces qui peuvent avoir sur nos systemes.

Mais avant de commencer de faire toute la pratique que nous aimont tant, je suis oblige de vous faire passer par un peu de theorie.

C’est quoi les regles et les decoders ?

Tout d’abord il faut savoir, que chaque ligne de log importe depuis un Wazuh-Agent, passe forcemment par un decoder, et au besoin elle passe egalement par une rule. Mais c’est quoi un Decoder ?

Un decoder c’est tout simplement une regex, que le Wazuh-Server va esayer de faire passer a chaque ligne de log, ces decoders permettent d’extraire tout type d’information a propos d’un log donnee.

Par exemple, une ligne de log SSH, peut ressembler a cela :

May 21 20:22:28 slacker2 sshd[8813]: Accepted password for root from 192.168.20.185 port 1066 ssh2
May 21 20:22:28 sol2 sshd[23857]: [ID 702911 auth.notice] User test1, coming from 192.168.2.185,  -  authenticated.
Oct 11 08:05:46 hostname auth|security:info sshd[323808]: Accepted publickey for usr1 from 2.3.4.5 port 37909 ssh2

Lorsque ce groupe de lignes de log va arriver vers notre Wazuh-Server, il va les faire passer une par une par des decoders (des regex) et recuperer des infos telles que :

  • Le Timestamp
  • Le serivce ssh
  • Le user qui s’authentifie
  • IP SRC
  • Erreurs

Et d’autres choses que ssh peut nous generer. Si on veut on peut voir ces decoders, il faudra juste aller vers : Server Management/Decoders

Vous pouvez voir d’un coeup d’oeil qu’il s’agit simplement de string regex, ou les logs seront passes un par un pour extraire tout type d’information.

Parlons maintenant des Rules, une fois que le Wazuh-Server, aura fait passer les logs par les decoders, il va faire passer ces chaines de caractere par les rules, c’est a dire qu’il va essayer de faire une corelation entre ce que les logs disent et un niveau d’alerte ou une Mitre ATTACK ou des niveau de compliance, ou tous a la fois.

Installation SYSMON

Sysmon si vous ne le connaisez pas, est un outil developpe par l’equipe sysinternals qui ont cree beaucoup d’utilitaires tres utiles pour les sysadmin. C’est egalement un outil tres puissant qui permet une meilleure journalisation des évènements de sécurité système sous Windows. Il s’agit d’un élément indispensable pour une surveillance efficace.

Sysmon permet entre autres de logger tous ces types d’evennements :

  • Création de processus
  • Connexion réseau
  • Fin d’un processus
  • Chargement de pilote
  • Chargement d’image
  • Evènement CreateRemoteThread (création d’un thread par un processus)
  • Evènement RawAccessRead
  • Requête DNS

Donc ce n’est pas un outil qu’il faut sousestimer, pour l’installer rien de plus simple on se dirige vers leur site officiel :

  • Lien vers site web sysinternals

Egalement il faudra telecharger cette configuration de sysmon :

  • Lien vers sysmon.xml

Cette configuration, elle fait partie d’un repository Github tres connu, qui va pre-parametrer notre sysmon avec toutes les alertes de base necessaires pour pouvoir detecter tout ce qui se passe dans une machine, tout ça en les mettant en relation avec des Mitre ATTACK, ce qui est tres utile.

Vous pouvez bien entendu creer votre propre fichier de configuration Sysmon, mais je ne vais pas m’attarder sur cela, pour faire que le post soit le plus concis possible.

Une fois que vous aurez telecharge votre executable sysmon il faudra l’extraire et lancer la commande suivante dans un Powershell avec les droit Admin :

sysmon.exe -accepteula -i sysmonconfig.xml

Avec cette commande, on va juste dire a Sysmon quel sont les choses qu’il doit surveiller (il y en a vraiment beaucoup) et une fois qu’il connait ça, il va se mettre en mode daemon et normalement demarrer automatiquement a tous les prochains redemarrages.

Ensuite je vous invite a aller au canal de journalisation suivant dans l’observateur d’evennements :

  • Journaux D'applications\Microsoft\Windows\Sysmon\Operational

Et vous verrez tous les logs que le sysmon aura deja receulli :

Modification Groupes Windows

Maintenant il faut bien entendu recuperer ces evennements, car a l’instant notre Wazuh-Agent, ignore que ce canal de journalisation existe, donc il faut lui dire d’aller les recuprer. Pour cela, on va se diriger vers la page des groupes,

Puis on appuie sur le bouton du stylo de notre groupe Windows :

Une fois dedans, je vous invite a copier coller ceci :

Group Windows Agent

Avec ce fichier de config qui sera deploye sur tous nos serveurs Windows on va pouvoir recuperer les config Sysmon, et autre en plus des informations qu’on a deja, par exemple on a active le module FIM, c’est a dire qu’on va pouvoir verifier l’integrite des fichiers, et voir si ils ont ete modifies.

On va egalement surveiller de pret le compertemment de certains .exe, surveiller certaines entrees Regedit qui sont sensibles, entre d’autres choses. C’est un fichier de config vraiment tres tres complet.

Une fois qu’on a deploye ce ficher de conf, on va maintenant importer les nouvelles regles sur notre Wazuh-Manager qui vont nous servir pour traiter ces nouvelles informations. Je vous donne ci dessous un script bash qu’il faut lancer sur le serveur ou se trouve le Wazuh-Manager, c’est un script qui va recuperer pas mal de regles faites par l’equipe SOC-Fortress, qui sont bien faites et tres optimises.

Pour les importer on lance ce script dans le serveur :

curl -so ~/wazuh_socfortress_rules.sh https://raw.githubusercontent.com/socfortress/Wazuh-Rules/main/wazuh_socfortress_rules.sh && bash ~/wazuh_socfortress_rules.sh
